## 🔥 The Problem

```
Most ed-tech is passive.
Watch video → Take test → Repeat → Still fail.

Students don't fail because they aren't smart.
They fail because:

  ❌  Study wrong topics at wrong time
  ❌  No feedback loop between effort and actual improvement
  ❌  No early warning for weak areas before exam day
  ❌  Coaching can't personalize to each individual student
  ❌  Zero connection between daily study and final score

StudyHive is built to fix all of this — one student at a time.
```

---

## 🧠 What is StudyHive?

StudyHive is a **full-stack AI academic operating system** powered by **ARIA** — *Adaptive Reasoning & Intelligence Architecture* — built on **Google Gemini 2.5 Flash**.

```
Student answers a question
          ↓
ARIA records → concept · difficulty · time spent · correct/wrong
          ↓
Updates   → Knowledge Graph · Error DNA · Forgetting Curve · XP · Streak
          ↓
Dashboard → real accuracy · real weak topics · real predictions
          ↓
ARIA acts → personalizes every question · every plan · every explanation
```

> Think: **Notion + Discord + Khan Academy + ChatGPT + Duolingo** —
> built specifically and obsessively for the Indian exam ecosystem.

## 🔌 API Reference

### Auth
```http
POST  /auth/register           Create account
POST  /auth/login              Sign in
GET   /auth/google             Google OAuth entry
POST  /auth/refresh            Refresh access token
GET   /auth/me                 Current user profile
```

### ARIA Intelligence
```http
POST  /api/aria/chat                Message → Gemini response
GET   /api/aria/daily-brief         Personalized morning brief
GET   /api/aria/knowledge-graph     Concept mastery map
GET   /api/aria/error-dna           Recurring mistake patterns
GET   /api/aria/score-prediction    Percentile forecast
POST  /api/aria/crash-plan          Emergency revision plan
POST  /api/aria/generate-mindmap    Chapter mind map
POST  /api/aria/process-pdf         PDF → exam summary
```

### Questions + Battles
```http
POST  /api/questions/answer         Submit answer → update graph
GET   /api/questions/practice       Calibrated practice set
POST  /api/battles/matchmake        Find ELO-matched opponent
GET   /api/battles/history          Past battles + ELO changes
GET   /api/battles/leaderboard      Global top players
```

### Analytics + Scheduler
```http
GET   /api/analytics/overview           All key metrics
```

## 🏗️ How It Was Built

```
Google Stitch  ──MCP──▶  Antigravity IDE  ──▶  GitHub  ──▶  Vercel + Railway
(UI Design)              (AI Coding)           (Code)        (Production)

1 → Generated screens in Google Stitch via Vibe Design prompts
2 → Connected Stitch to Antigravity via MCP — fetched Design DNA (34 colors)
3 → AI agent converted Stitch HTML → React components
4 → Upgraded every element with ShadCN replacements
5 → Added complete GSAP animation layer
6 → Built Node.js backend with Gemini AI integration
7 → Implemented full auth system + MongoDB schemas
8 → Deployed pipeline ready for Vercel + Railway
```
